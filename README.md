# Мини-бот Telegram с контекстом

## Краткий отчет по проекту

- Реализована интеграция с LLM `gpt-5.4-mini` через `openai` и прокси-эндпоинт `https://api.proxyapi.ru/openai/v1`.
- Реализован пользовательский контекст диалога в памяти (`context_manager.py`) с раздельным хранением по `user_id`.
- Добавлены команды управления диалогом: `/reset`, `/temp=<число>`, `/tokens=<число>`.
- Входящие сообщения, не начинающиеся с `/` или `!`, отправляются в модель; ответы модели возвращаются пользователю.
- Реализовано логирование общения с моделью: дата/время, пользователь, `temperature`, `max_tokens`, входящие и выходящие токены.

### Краткие выводы по влиянию параметров хапросов к модели

- Параметр max_tokens при небольшом значении может обрезать ответ модели. Модель хотела составить перечень 10 математических парадоксов (запрос 6), но обрезала ответ посреди восьмого.
- Не замечено значительного влияния параметра temperature на креативность ответа модели.

## Установка, настройка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/kostanov/context-bot.git
cd context-bot
```

### 2. Установка зависимостей

Рекомендуется использовать [uv](https://github.com/astral-sh/uv) для ускоренной установки:

```bash
uv sync
```

### 3. Переменные окружения

Создайте файл `.env` в корне проекта (или скопируйте его из .env.example) со следующим содержимым:

```
BOT_TOKEN=ваш_токен_бота
PROXY_API_KEY=ваш_ключ_ProxyAPI
# (опционально)
# BOT_BASE_URL=кастомный_адрес_telegram_api
# BOT_BASE_FILE_URL=кастомный_адрес_file_api
# LLM_LOG_FILE=llm_chat.log
```

- `BOT_TOKEN` — токен Telegram-бота, его можно получить через BotFather.
- `PROXY_API_KEY` — API-ключ сервиса [proxyapi.ru](https://proxyapi.ru/).
- Остальные переменные используются только при необходимости специфического API/логирования.

### 4. Запуск бота

Для запуска используйте uv:

```bash
uv run context-bot
```

или утилиту [just](https://github.com/casey/just)

```bash
just run
```

Бот запустится и начнет принимать сообщения. Для остановки используйте `Ctrl+C`.

### 5. Просмотр логов

Все обращения к LLM логируются в файл, указанный в переменной `LLM_LOG_FILE` (по умолчанию — `llm_chat.log`).
